А можно в стратегии ли использовать индикаторы, описанные в статье
CORRELATION AS A CYCLE INDICATOR как фильтры, или как дополнительные
сигналы для индикатора The AutoTune Filter? Есть ли в этом смысл,
если да- объясни в чем он и как правильно совместить эти индикаторы?


Да, **можно**, и в этом есть смысл. Но я бы использовал индикаторы из `CORRELATION AS A CYCLE INDICATOR` **не как ещё один равноправный генератор входов**, а как:

1. **фильтр режима рынка** — рынок сейчас циклический или трендовый;
2. **подтверждение фазы цикла** — AutoTune говорит “похоже, разворот”, а Correlation Cycle проверяет, действительно ли мы в нужной части цикла;
3. **запрет плохих сделок** — особенно запрет шортов в сильном восходящем режиме и лонгов в нисходящем.

Моя экспертная оценка: **совмещать имеет смысл**, но аккуратно. Оба индикатора относятся к циклическому анализу Элерса, поэтому если просто сложить их сигналы по принципу “сработал любой — входим”, получится не усиление, а шум и переоптимизация.

---

## В чём разница между AutoTune Filter и Correlation Cycle

### The AutoTune Filter

AutoTune делает примерно следующее:

1. очищает цену через Highpass;
2. ищет доминирующий цикл через автокорреляцию;
3. подстраивает Bandpass-фильтр под найденный цикл;
4. использует разворот Bandpass-линии как сигнал.

В статье стратегия примерно такая:

```python
long_signal = ROC пересекает 0 вверх and MinCorr < Threshold

short_signal = ROC пересекает 0 вниз and MinCorr < Threshold and Filt > 0
```

То есть AutoTune — это в первую очередь **адаптивный ловец циклических разворотов**.

---

### Correlation as a Cycle Indicator

В этой статье Элерс строит другой набор понятий:

| Элемент | Простое объяснение                                |
| ------- | ------------------------------------------------- |
| `Real`  | насколько цена похожа на косинусную волну         |
| `Imag`  | насколько цена похожа на сдвинутую синусную волну |
| `Angle` | фаза цикла: где мы сейчас внутри волны            |
| `State` | состояние рынка: цикл, тренд вверх, тренд вниз    |

Очень простая идея такая:

* если рынок в **cycle mode**, можно торговать от впадины к пику и от пика к впадине;
* если рынок в **trend mode**, обычные циклические развороты начинают ломаться;
* провал циклической модели сам становится сигналом, что рынок перешёл в тренд.

Вот это как раз полезно для AutoTune: AutoTune ловит развороты, а Correlation Cycle может сказать:
**“сейчас вообще безопасно ловить циклический разворот или рынок уже ушёл в тренд?”**

---

## Самый правильный способ совмещения

Я бы начал с такой архитектуры:

```text
AutoTune Filter = основной генератор входов
Correlation Cycle = фильтр режима и фазы
```

То есть AutoTune решает:

> “есть ли технический сигнал на вход?”

А Correlation Cycle проверяет:

> “имеет ли этот сигнал смысл в текущем состоянии рынка?”

---

## Вариант 1. Консервативная mean-reversion стратегия

Это самый логичный вариант.

### Лонг

Входить в long только если:

```text
AutoTune даёт long-сигнал
И
MinCorr достаточно отрицательный
И
Correlation Cycle показывает cycle mode
И
Angle < 0
```

Простыми словами:

> AutoTune говорит: “похоже, начинается движение вверх от впадины”.
> Correlation Cycle подтверждает: “да, рынок сейчас похож на цикл, и мы действительно в восходящей половине цикла”.

### Шорт

Входить в short только если:

```text
AutoTune даёт short-сигнал
И
MinCorr достаточно отрицательный
И
Correlation Cycle показывает cycle mode
И
Angle > 0
```

Простыми словами:

> AutoTune говорит: “похоже, начинается движение вниз от пика”.
> Correlation Cycle подтверждает: “да, рынок циклический, и мы в нисходящей половине цикла”.

---

## Почему это имеет смысл

Проблема AutoTune в том, что он может хорошо ловить развороты в циклическом рынке, но в тренде такие развороты часто становятся преждевременными.

Например:

```text
Цена растёт сильным трендом.
AutoTune видит локальный пик.
Он может дать short-сигнал.
Но рынок не разворачивается, а просто делает маленькую паузу и идёт дальше вверх.
```

Correlation Cycle может помочь отфильтровать такую ситуацию:

```text
State = +1 → рынок в режиме тренда вверх
```

Тогда short лучше не брать.

---

## Вариант 2. Correlation Cycle как запрет против тренда

Это, на мой взгляд, даже важнее, чем подтверждение входов.

Правила можно сделать так:

```text
если State = +1:
    запрещаем short-сигналы AutoTune
    long-сигналы можно разрешить, но осторожно

если State = -1:
    запрещаем long-сигналы AutoTune
    short-сигналы можно разрешить

если State = 0:
    торгуем обычные циклические сигналы AutoTune
```

То есть `State` используется как **режимный фильтр**.

Пример:

```python
if autotune_long and corr_state != -1:
    buy()

if autotune_short and corr_state != +1:
    sell_short()
```

Но ещё лучше — строже:

```python
if autotune_long and corr_state == 0 and corr_angle < 0:
    buy()

if autotune_short and corr_state == 0 and corr_angle > 0:
    sell_short()
```

Первый вариант мягче, второй вариант строже.

---

## Вариант 3. Разделить циклический и трендовый режим

Можно сделать стратегию с двумя режимами.

### Если `State == 0`

Рынок циклический. Тогда используем AutoTune как mean-reversion:

```text
ROC AutoTune пересёк 0 вверх → long
ROC AutoTune пересёк 0 вниз → short / exit long
```

### Если `State == +1`

Рынок в тренде вверх. Тогда:

```text
не шортим;
long можно держать дольше;
выходить только по более сильному признаку разворота.
```

### Если `State == -1`

Рынок в тренде вниз. Тогда:

```text
не лонгуем;
short можно держать дольше;
выходить только по более сильному признаку разворота.
```

Это уже более сложная, но более “рыночная” логика.

---

## Что НЕ надо делать

Я бы не делал так:

```text
AutoTune дал long ИЛИ Correlation Cycle дал long → покупаем
```

Это плохая идея.

Почему:

* оба индикатора анализируют циклы;
* они не являются полностью независимыми источниками информации;
* при логике `ИЛИ` количество сделок вырастет;
* качество сделок, скорее всего, упадёт;
* стратегия начнёт входить слишком часто.

Также не стоит делать простую голосовалку:

```text
если 2 из 3 индикаторов за long → покупаем
```

Для Элерса это обычно не лучший путь. Здесь важнее не “сколько индикаторов согласно”, а **какую роль выполняет каждый индикатор**.

---

## Как бы я совместил на практике

Я бы начал с такой версии:

```text
1. AutoTune ищет точку входа.
2. Correlation Cycle определяет режим рынка.
3. Если рынок циклический — торгуем развороты AutoTune.
4. Если рынок трендовый — запрещаем сделки против тренда.
5. Если индикаторы противоречат друг другу — пропускаем сделку.
```

В виде логики:

```python
autotune_long = atf.roc crosses above 0 and atf.mincorr < threshold
autotune_short = atf.roc crosses below 0 and atf.mincorr < threshold

cycle_mode = corr.state == 0
trend_up = corr.state == 1
trend_down = corr.state == -1

phase_long_ok = corr.angle < 0
phase_short_ok = corr.angle > 0

long_signal = (
    autotune_long
    and cycle_mode
    and phase_long_ok
)

short_signal = (
    autotune_short
    and cycle_mode
    and phase_short_ok
)
```

Это базовая версия.

Потом можно добавить трендовый вариант:

```python
long_signal = (
    autotune_long
    and (
        (cycle_mode and phase_long_ok)
        or trend_up
    )
)

short_signal = (
    autotune_short
    and (
        (cycle_mode and phase_short_ok)
        or trend_down
    )
)
```

Но я бы начинал именно с первой, более строгой версии.

## Важный момент: какой Period брать для Correlation Cycle

В статье `CORRELATION AS A CYCLE INDICATOR` используется фиксированный `Period`, например 20 или 14.

А AutoTune сам вычисляет `DC` — Dominant Cycle.

Поэтому есть два варианта.

### Вариант А. Фиксированный Correlation Period

Например:

```text
AutoTune: Window = 20–26
Correlation Cycle: Period = 14 или 20
```

Плюс: Correlation Cycle остаётся более независимой проверкой.

Минус: если реальный цикл сильно меняется, фильтр может запаздывать или ошибаться.

---

### Вариант Б. Использовать `DC` из AutoTune

Например:

```text
Correlation Period = round(AutoTune DC)
```

Плюс: оба индикатора смотрят на один и тот же доминирующий цикл.

Минус: они становятся менее независимыми, потому что Correlation Cycle уже зависит от AutoTune.

Мой выбор для первого теста:

```text
Сначала фиксированный Period = 20.
Потом отдельным экспериментом — Period = AutoTune DC.
```

Так будет честнее при бэктесте: ты увидишь, даёт ли Correlation Cycle реальную пользу как независимый фильтр.

---

## Что подавать на вход Correlation Cycle

Здесь тоже есть два варианта.

### Вариант 1. Подавать обычный Close

Это лучше, если ты хочешь использовать Correlation Cycle как **режимный фильтр**:

```text
Close → Correlation Cycle → State / Angle
```

Он будет смотреть на рынок в целом и пытаться понять: цикл сейчас или тренд.

---

### Вариант 2. Подавать очищенную линию AutoTune

Например:

```text
AutoTune Filt или BP → Correlation Cycle
```

Это лучше, если ты хочешь использовать Correlation Cycle как **подтверждение фазы** именно для AutoTune.

Но для начала я бы выбрал первый вариант:

```text
AutoTune работает по Close.
Correlation Cycle тоже работает по Close.
Correlation Cycle используется как фильтр.
```

Так проще интерпретировать результат.

---

## Как проверять, есть ли от этого польза

Нужно сравнить минимум 4 версии стратегии:

| Версия | Что проверяем                               |
| ------ | ------------------------------------------- |
| 1      | Только AutoTune                             |
| 2      | AutoTune + запрет против `State`            |
| 3      | AutoTune + `State == 0` + проверка `Angle`  |
| 4      | AutoTune + Correlation Period = AutoTune DC |

Сравнивать нужно не только прибыль.

Смотри:

* количество сделок;
* profit factor;
* max drawdown;
* среднюю сделку;
* отдельно long и short;
* сколько сигналов было отфильтровано;
* стала ли equity curve ровнее.

Если после добавления Correlation Cycle сделок стало меньше, прибыль чуть ниже, но просадка сильно меньше и profit factor выше — это может быть хорошее улучшение.

Если сделок стало меньше, прибыль меньше, просадка почти такая же — фильтр бесполезен.

---

## Мой вывод

Да, использовать индикаторы из `CORRELATION AS A CYCLE INDICATOR` вместе с `The AutoTune Filter` **имеет смысл**.

Но правильная роль такая:

```text
AutoTune = основной сигнал входа
Correlation Cycle = фильтр режима рынка и подтверждение фазы
```

Самая разумная первая версия:

```text
Long:
AutoTune ROC пересёк 0 вверх
+ MinCorr < Threshold
+ Correlation State == 0
+ Correlation Angle < 0

Short:
AutoTune ROC пересёк 0 вниз
+ MinCorr < Threshold
+ Correlation State == 0
+ Correlation Angle > 0
```

А затем отдельно протестировать более мягкую версию, где при `State == +1` разрешены только long, а при `State == -1` разрешены только short.

Главная идея: **Correlation Cycle не должен дублировать AutoTune. Он должен отвечать на другой вопрос: “сейчас вообще подходящий режим рынка для этого сигнала?”**
